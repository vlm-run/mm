  Bugs found in docs/BENCHMARK-FIXTURES.md                                                                                                                                                                                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                                                                                                                                                                                  
  1. Wrong flag name everywhere: doc says --encoder.strategy, real flag is --encode.strategy.                                                                                                                                                                                                                                                                                                                                                                   
  2. Wrong encoder names: doc uses image-tiled, resize, ocr.                                                                                                                                                                                                                                                                                                                                                                                                      
    - image-tiled → should be tile                                                                                                                                                                                                                                                                                                                                                                                                                              
    - resize is valid                                                                                                                                                                                                                                                                                                                                                                                                                                             
    - ocr doesn't exist; closest alternatives are page-text, rasterize, rasterize-text.                                                                                                                                                                                                                                                                                                                                                                         
  3. Audio fast description wrong: doc says "w/ whisper.tiny" and "with whisper.medium" for accurate — but fast actually runs whisper_model=medium per --list-encoders default and confirmed in output (model=medium, 31377ms). Fast/accurate use the same Whisper model on audio.                                                                                                                                                                                
  4. Non-PDF docx fixture (input.docx -m accurate) — no sample file exists in mmbench-tiny to validate; fixture is undemonstrable against provided datasets.                                                                                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                                                                                                                                                                                                  
  Bugs found in the CLI itself                                                                                                                                                                                                                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                                                                                                                                                                                                                  
  1. --list-encoders name mismatch. The panel displays image-tile, image-resize, video-frame-sample, document-page-text, etc., but resolve_strategy only accepts the bare names (tile, resize, frame-sample, page-text). Users copy the displayed name → KeyError. Fix: either register canonical kind-prefix names or print the bare registry key.                                                                                                               
  2. Encoder override only partially honored by pipelines.                                                                                                                                                                                                                                                                                                                                                                                                        
    - --encode.strategy audio-gemini -m accurate still runs Whisper (audio_transcription_ms: 28930ms in the metrics footer). The accurate audio pipeline appears to force-transcribe regardless of encoder.                                                                                                                                                                                                                                                       
    - --encode.strategy frame-sample -m accurate on video still reports mosaic_assembly_ms and scene_detection_ms — the accurate video pipeline hard-wires mosaic stages and isn't replaced by the flag.                                                                                                                                                                                                                                                          
    - Net: --encode.strategy on accurate pipelines is silently ignored for audio/video.                                                                                                                                                                                                                                                                                                                                                                           
  3. PDF accurate output truncated. Last cell ends $1 with no newline/closing. Likely a max-tokens cap hit on the last page batch — should either bump max_tokens on the final batch or warn.                                                                                                                                                                                                                                                                     
  4. Video fast stderr noise. objc[...]: Class AVFFrameReceiver is implemented in both cv2 and av dylibs — needs cleanup (likely unload unused ffmpeg backend, or suppress class duplication via PYAV_LOGGING).                                                                                                                                                                                                                                                   
  5. Fast image default dumps raw base64 JSON to stdout (66KB for a 39KB JPEG). Not useful for humans in a TTY. Should be rich-panel by default and only emit JSON when piped or --format json.                                                                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                                                                                                                                                                                                                                
  Feature analysis / suggested improvements                                                                                                                                                                                                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                                                                                                                                                                                                  
  Throughput (tok-oriented):                                                                                                                                                                                                                                                                                                                                                                                                                                      
  - Image fast: ~130KB/s encode. Negligible — dominated by startup.                                                                                                                                                                                                                                                                                                                                                                                               
  - PDF fast: 8 pages / 41ms ≈ 195 pages/s via pypdfium2. Excellent.                                                                                                                                                                                                                                                                                                                                                                                              
  - PDF accurate: 8 pages / 45.7s = 0.17 pg/s, 15.3s startup + VLM. Batch=4 per call → 2 calls × ~22s. Mostly VLM latency.                                                                                                                                                                                                                                                                                                                                        
  - Audio fast (24 min / 14 MB → 31.4s Whisper medium): realtime ≈ 46×, ~0.44 MB/s.                                                                                                                                                                                                                                                                                                                                                                               
  - Audio accurate: 52.8s wall for 24 min → 27× realtime.                                                                                                                                                                                                                                                                                                                                                                                                         
  - Video fast (4m13s → 6.3s): 40× realtime, 4.6 MB/s.                                                                                                                                                                                                                                                                                                                                                                                                            
  - Video accurate: 15.3s wall / 253s duration → 16.5× realtime; VLM 9.2s dominates.                                                                                                                                                                                                                                                                                                                                                                              
                                                                                    
  Concrete improvement suggestions                                                                                                                                                                                                                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                                                                                                                                                                                                                
  1. Fix doc to match reality. Run the fixtures in CI against mmbench-tiny as a smoke test so the doc can't drift.                                                                                                                                                                                                                                                                                                                                                
  2. Unify encoder naming. Either make the registry key {kind}-{name} (matching the displayed name) or drop the prefix in display. Pick one, document one.                                                                                                                                                                                                                                                                                                        
  3. Make --encode.strategy authoritative. When provided, the pipeline's encode stage should be replaced by the specified encoder, including skipping Whisper for audio-gemini and mosaic for frame-sample. Current behavior silently ignores the override inside accurate pipelines.                                                                                                                                                                             
  4. Default TTY output for fast image/video should be human-readable. Print a small summary panel (dims, bytes, base64 len, pipeline spec) and only emit JSON when piped or -o json.                                                                                                                                                                                                                                                                             
  5. Whisper model should be tiered fast/accurate. Fast path should run tiny (or base) for 5-10× speedup; accurate → medium. Fixture text promises this but code does not.                                                                                                                                                                                                                                                                                        
  6. Add a real ocr encoder (tesseract or easyocr or pypdfium2+tesseract fallback) — many scanned PDFs return empty text from pypdfium2 and silently produce no content today.                                                                                                                                                                                                                                                                                    
  7. Bound VLM output in PDF accurate per-batch so the last-batch truncation ($1) can't happen, or detect finish_reason=length and warn.                                                                                                                                                                                                                                                                                                                          
  8. Silence dylib warnings. Either drop opencv-python (use av only) or wrap stderr to filter the objc[...] lines from non-JSON modes — they currently leak into stdout captures when users pipe.                                                                                                                                                                                                                                                                 
  9. Report per-stage metrics consistently. Image-accurate omits the [mode=accurate, total=…] footer that audio/video include — standardize so every accurate run has the same trailer for benchmarking.                                                                                                                                                                                                                                                          
  10. Add a --dry-run / --summary mode that shows the resolved pipeline (pipeline file, encode stage with params, generate stage) and exits — very helpful given current override ambiguity.
